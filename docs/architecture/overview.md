# 🏗️ 生态系统与整体架构

ECMDB 现已从演进为微服务生态架构，涵盖四个独立系统，打通从 **监控告警 -> 资产管理 -> 工单审批 -> 自动执行** 的闭环体系。

## 🧩 核心组件生态全览

- 🖥️ **ECMDB (基础元数据与流程后台)**
  - **组件定位**：提供系统核心数据及控制逻辑。
  - **核心功能**：
    - **CMDB 资源管理**：使用自定义模型来管理物理机、云主机、K8S 集群等资产，并维护资源拓扑关系。支持基于 ngram 的全局资产模糊搜索。
    - **工单流转控制**：对操作申请进行标准化管理。支持会签、或签及条件分支，可与飞书/企业微信对接消息及审批流。内置 Casbin 权限体系。
    - **排班调度计算**：提供支持多种 RRULE 循环规则的排班计算逻辑，提供排班表用于后续告警信息的分发对接。
  - **仓库地址**：[🔗 访问 GitHub 源码](https://github.com/Duke1616/ecmdb)

- 🎨 **ECMDB-Web (应用前端台)**
  - **组件定位**：为操作人员提供可视化管理界面。
  - **核心功能**：
    - **资产关系可视化**：将平级的资产明细转化为清晰的关系图谱，支持下钻查看资源的关联节点拓扑。
    - **工作流可视化编排**：内置了基于节点的拖拽式工作流编辑器，并搭载动态表单设计器，便于非代码方式快捷搭建流程。
    - **一体化运维视图**：提供内置日历组件管理排班信息，并在界面直接嵌入 Web Terminal（基于 xterm.js）方便直接连接资产进行操作。
  - **仓库地址**：[🔗 访问 GitHub 源码](https://github.com/Duke1616/ecmdb-web)

- 🔔 **EAlert (监控告警引擎)**
  - **组件定位**：对接并处理上游监控数据的独立处理网关。
  - **核心功能**：
    - **告警事件收敛**：接入 Prometheus 等多源监控报警，基于设定的路由规则聚合与去重，降低重复告警对维护人员的打扰。
    - **精准通知路由**：告警触发后，通过关联 ECMDB 的排班数据，直接路由通知给当前确切的负责人员。
    - **告警转工单联动**：在检测到特定故障时，支持将告警信息派发到 ECMDB 创建异常排障工单流转。
  - **仓库地址**：🔒 *核心组件，暂未开源*

- ⚙️ **ETask (分布式异步执行单元)**
  - **组件定位**：部署在独立网络中的执行器。
  - **核心功能**：
    - **跨网络执行**：负责承载核心后台下发的跨网段和耗时自动化剧本（如：并发数据巡检或系统指令）。
    - **主应用解耦**：利用 Kafka 作为消息队列将耗时任务与 ECMDB 后台异步隔离，使得前端大面积拉取任务及资源探测时免受阻塞影响，保证主控界面的响应能力。
  - **仓库地址**：[🔗 访问 GitHub 源码](https://github.com/Duke1616/etask)

## 🛠️ 集成技术选型

### 后端与中间件层
- **开发语言**：基于 **Go 1.25** 构建，使用协程应对并发。
- **存储方案**：**Percona MongoDB**（提供动态模型扩展字段存储并自带全文检索） + **MySQL 8.x**（管理关联控制型核心业务数据）。
- **缓存与中间件**：**Redis**（缓存支持与锁状态） + **Kafka** & **Etcd**（保障 ETask 的服务注册与异步消息削峰）。

### 前端应用层
- **基础栈**：Vite + **Vue 3** (Composition API) + TypeScript
- **状态及界面**：Pinia 全局持久化状态管理 + Element Plus 组件库
- **核心依赖库**：集成 `@logicflow/core` 控制拖拽流程，`@form-create` 生成动态表单，`relation-graph` 处理资产拓扑展现。
