# 模版管理

模版定义了工单的前端表单结构及关联的流程配置。用户通过模版提交业务请求，系统根据配置初始化数据并触发对应的后端流程。

## 1. 核心属性

- **业务分组**：对模版进行逻辑分类，直接决定门户页的展示布局与访问入口。
- **绑定流程**：指定该模版触发的后端工作流逻辑。
- **收藏功能**：用户可将高频使用的模版收藏至个人工作台，提升提单效率。

## 2. 配置说明

<img src="/images/manager/template.png" style="width: 100%; border-radius: 8px; margin: 16px 0;" alt="模版管理预览" />

模版配置采用分步导向模式，分为基础信息与表单设计两个阶段：

### 基础配置
- **名称与描述**：定义模版展示标题及提单指引文档。
- **图标选择**：配置门户页展示图标。
- **所属分组**：关联对应的业务分类。
- **工作流关联**：指定该模版绑定的流程定义 ID。

### 表单定义
系统集成 **FormCreate** 与 **FcDesigner** 实现可视化的表单编排：
- **动态组件**：支持文本、下拉框、日期、人员选择等多种标准表单组件。
- **字段 Key (字段ID)**：**全局唯一标识码**。定义字段在全链路中的数据识别码。
- **引用逻辑**：
    - **逻辑配置**：在工作流审批节点（按字段找人）或自动化节点（动态延时）中，**直接填写 key** 即可实现逻辑关联。
    - **文案替换**：在节点通知标题或群消息中，可使用 <span v-pre>`{{field.key}}`</span> 进行文本动态渲染。
    - **脚本访问**：自动化脚本通过运行时的 `args` 上下文（JSON）获取表单数据，而非代码级别的模版解析。

## 3. 运行逻辑

- **数据快照**：工单发起时，系统会创建当前模版定义的静态快照。后续对模版的任何修改均不影响已存在的历史工单实例。
- **结构一致性**：更新表单字段 ID 时，需同步维护关联流程中涉及该 ID 的变量引用。

---

> [!IMPORTANT]
> **设计准则**：建议在模版说明中明确 SLA 或前置检查项，通过规范化的输入减少后续流程的人工干预成本。
