# 🏁 开始节点 (Start Node)

开始节点是流程的唯一入口。在 ECMDB 中，它负责初始化工单实例并设置初识状态。

## 📝 核心配置

### 1. 基础信息
- **节点名称**：用于在流程图中标识入口。
- **消息通知 (`is_notify`)**：开启后，工单发起成功将立即向触发人推送 **飞书交互卡片**。
    - **特权**：允许用户直接在社交侧 **实时查看处理进度** 或执行 **一键撤回** 操作，实现极简的移动端办结体验。

### 2. 人员记录
- **$starter**：系统默认将发起人标记为 `$starter`，作为后续流程变量的基础。

### 3. 数据层级
- 开始节点绑定的表单数据会持久化到工单的中。
- 这些字段在整个生命周期内全局可见。

## 💡 技术细节

### EventStart (流程启动)
在后端逻辑中，开始节点被赋予 `NodeType: 0`。该节点在转换时会自动注册 `EventStart` 事件：

**核心职责：**
1. **工单与流程绑定**：将业务侧的 `Order ID` 与流程引擎的 `Instance ID` 进行强关联，确立后续流转的主键基础。
2. **状态自动流转**：触发工单状态由 **「待处理 (START)」** 变更为 **「进行中 (PROCESS)」**。
3. **下发集成卡片**：开启 `is_notify` 后，该事件负责向发起人推送第一条 Feishu 交互卡片，提供进度查询与撤回入口。

```go
n := model.Node{
    NodeID: node.ID, 
    NodeType: 0, 
    UserIDs: []string{"$starter"},
    NodeEndEvents: []string{"EventStart"},
}
```

---

> [!TIP]
> 建议在开始节点配置清晰的名称（如：资产申领入口），便于在审计日志中回溯。
