# 开始节点

开始节点是流程的唯一入口，负责工单实例的初始化、初始状态设置以及全局变量的定义。

## 1、核心配置

### 基础信息
- **节点名称**：用于在流程图中标识业务入口。
- **消息通知**：开启该选项后，工单发起成功将立即向发起人推送飞书动态卡片。
    - **交互功能**：发起人可通过卡片直接查看处理进度或执行撤回操作，实现业务流程的主动触达。

### 人员定义
- **发起人**：系统默认将发起人标识为内置变量 `$starter`，作为后续流程中查找处理人（如“直属领导”、“工单发起人”）的基础。

### 数据隔离
- 流程启动时提交的表单数据会持久化至工单全局上下文。
- 相关字段在流程完整生命周期内全局可见，并可被下游自动化节点或网关引用。

## 2、执行细节

### EventStart 事件
在流程引擎中，开始节点通过注册 `EventStart` 事件来驱动后续逻辑：

1.  **实例绑定**：完成业务工单（Order ID）与流程引擎实例（Instance ID）的关联，建立流转的主键基础。
2.  **状态变更**：触发工单状态由 **「已提交」** 自动切换为 **「流转中」**。
3.  **推送初始化**：若开启消息通知，该事件将触发首条飞书交互消息推送，为发起人提供查询与撤回的入口。

---

> [!TIP]
> **最佳实践**：建议为开始节点指定明确的业务名称（如：权限申请入口），有助于在流程审计和流转历史中快速定位。
