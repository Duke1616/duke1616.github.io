# 🔀 网关节点 (Gateways)

网关节点是控制流程分支逻辑的枢纽。ECMDB 支持多种符合 BPMN 标准的网关类型。

## 📍 常见网关类型

### 1. 条件网关 (`condition`)
- **排他性**：流程会根据连线上的 `expression` (如 `amount > 100`) 进行判定。
- **匹配即止**：只会走入第一个符合条件的路径。

### 2. 并行网关 (`parallel`)
- **分流**：同时启动所有下级分支。
- **汇聚**：等待所有分支全部完成后，流程才会继续向后推进。

### 3. 包容网关 (`inclusion`)
- **灵活分流**：同时启动所有符合条件的路径（如果没有连线条件，则全部启动）。
- **异步汇聚**：只要有分支到达即可触发后续（取决于具体配置）。

### 4. 条件并行网关 (`selective`)
- 结合了条件判断与并行执行的能力。

## 📐 逻辑表达式

在连线属性中配置：
- 支持标准 SQL 风格或逻辑运算表达式。
- 示例：`status == 'fail' && retry_count < 3`

---

> [!CAUTION]
> 在配置并行/包容网关时，请确保每一条分支最终都能交汇或到达结束节点，避免流程产生孤立实例。
