# 📢 群通知节点 (Chat Node)

群通知节点（Chat）专门用于将实时进度推送到飞书或企业微信群聊。

## 📨 运行模式 (`mode`)

### 1. 现有群组 (`existing`)
- 通过 `chat_group_ids` 列表选择已在系统中注册的群聊机器人。
- 适用于「运维报警群」、「版本发布通知群」等固定场景。

### 2. 动态创建 (`create`)
- 流程运行到此时动态创建一个新的协作群。
- 可以根据 `Assignees` 规则自动拉取相关干系人进入群聊。

## 🖋️ 消息卡片定制 (`output_mode`)

您可以自定义消息卡片中展示哪些板块的内容（支持多选）：

- **工单提交信息 (`ticket_data`)**：展示开始节点用户填写的原始申请内容。
- **自动化任务结果 (`auto_task`)**：展示 Task 节点执行后的关键输出或状态。
- **用户节点信息 (`user_input`)**：展示中间环节审批人留下的审批意见或追加数据。

## 💡 使用场景

- **流程启动广播**：让整个部门知道有一个高风险发布开始了。
- **结果同步**：数据库变更成功后，自动将变更记录推送到 DBA 审计群。

---

> [!NOTE]
> 该节点为 **广播型节点**，发送消息后系统会瞬间完成并流向下一节点，不产生人工阻塞。
