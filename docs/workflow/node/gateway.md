# 🔀 网关节点

网关节点是流程的“导航员”，它负责根据您设定的条件，决定流程往哪条路走、是同时走多条路还是等大家都到齐了再走。

## 📍 核心类型

系统提供了四种逻辑，覆盖了几乎所有的运维分流场景：

### 1. 条件网关 (要么A，要么B)
- **排他性**：它像是一个单项选择题。
- **匹配即止**：流程会按顺序检查连线上的条件（比如：`金额 > 100`），只要找到第一个符合条件的，就顺着那条路走，剩下的路就不看了。

### 2. 并行网关 (大家一起动)
- **分流**：流程流转到这里，会**同时启动**所有的下级分支，不需要判断条件。
- **汇聚**：如果多条路最终回到一个并行网关，系统会**等所有人点同意**之后，才继续开启后面的任务。

### 3. 包容网关 (只要符合的都动)
- **灵活分流**：它会把连线上所有符合条件的路径**全部启动**。如果有三条路都满足条件，那就三条路并发执行。
- **异步处理**：非常适合复杂的竞争或非全员汇聚场景。

### 4. 则并网关 (智能分流)
- 这种网关结合了“条件判断”和“并行执行”，是针对特定复杂场景的进阶逻辑。

## 📐 条件表达式怎么写？

在两条节点之间的连线上，您可以配置判定逻辑：
- **通俗写法**：直接用 `==`（等于）、`!=`（不等于）、`>`（大于）等符号。
- **例子**：`环境 == '生产'`，那么只有当申请单里填的是生产环境时，才会走这条分支。

---

> [!CAUTION]
> **防止孤岛**：配置并行/包容网关时，请确保每一条分支最终都能汇总回来或者到达结束节点，否则可能会导致任务“卡死”在半路。
