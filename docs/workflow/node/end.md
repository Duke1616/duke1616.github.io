# 结束节点

结束节点标志着工单完整生命周期的终结。

## 结算逻辑

当流程到达结束节点后，系统将执行以下核心操作：

1.  **状态更迭**：工单业务状态由“流转中”正式变更为 **“已办结”**。
2.  **结果触达**：触发办结通知事件，向相关干系人反馈流程执行的最终结果。
3.  **数据封存**：锁定工单全局上下文变量池，**不再允许任何修改**，确保审计记录的严肃性。
4.  **流水存档**：将整个流程的审批轨迹、自动化执行日志进行归档存储，供后续追溯。

## 执行细节

在系统底层，结束节点负责执行最终的状态定格：

- **实例变迁**：更新数据库中的工单标志位，释放关联资源。
- **上下文快照**：将动态变量转化为静态的历史快照。

---

> [!NOTE]
> **多终点支持**：一个业务流程可以定义多个结束节点，用以代表不同的业务终点（例如：审批驳回致终结、自动化执行成功致办结等）。
