# 🏁 结束节点

结束节点标志着工单完整生命周期的终结。

## 💾 结算操作

当流程到达结束节点后，系统会执行以下核心操作：

1. **状态变迁**：工单状态由“处理中”正式变更为 **“已办结”**。
2. **通知触达**：触发通知事件，告知发起人他的诉求已经有了最终结果。
3. **数据封存**：锁定所有的工单口袋数据，**不再允许任何修改**，确保审计的严肃性。
4. **存档与审计**：将整个流程的审批轨迹、自动化执行日志全部打包存储，供日后追溯。

## ⚙️ 技术定义

在系统底层，结束节点负责执行最后的“握手”动作：

- **状态锁定**：修改数据库中的工单标志位。
- **快照定格**：将动态变量转化为静态的历史记录。

---

> [!NOTE]
> **多结局支持**：一个流程可以有多个结束节点，代表不同的业务终点（如：由于审批被驳回而结束、由于脚本执行成功而结束）。
