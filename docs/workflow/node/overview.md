# 🧩 节点总览

工单流程由一系列功能各异的节点构成。通过拖拽这些节点，您可以快速地将业务运维逻辑转换为自动化的工作流。

## 📍 节点分类

根据设计器中的功能划分，节点主要分为以下几类：

### 🚥 网关节点 (Gateway Nodes)
用于控制流程的流转方向，决定「下一步走哪里」。

| 节点名称 | 内部类型 | 功能描述 |
| :--- | :--- | :--- |
| **条件** | `condition` | **排他网关**：根据连线上的表达式进行判断，仅走入第一个匹配的分支。 |
| **并行** | `parallel` | **并行执行**：同时启动所有下级分支，并等待所有分支全部完成后再继续。 |
| **则并** | `selective` | **条件并行**：结合了表达式判定与并行执行的能力，仅启动符合条件的分支。 |
| **包容** | `inclusion` | **包含网关**：启动所有符合条件的分支，通常用于灵活的异步流转场景。 |

### 🏢 业务节点 (Business Nodes)
用于执行具体的业务逻辑，决定「下一步做什么」。

| 节点名称 | 内部类型 | 功能描述 |
| :--- | :--- | :--- |
| **用户** | `user` | **人工参与**：包含审批、抄送等需要人工干预的任务，支持多级规则设置。 |
| **自动化** | `automation` | **自动执行**：系统自动调用 ETask 脚本或 API，无需人工干预。 |
| **群通知** | `chat` | **外部推送**：向飞书/企业微信机器人发送通知卡片，支持实时进度共享。 |

### 🏁 系统节点 (System Nodes)
定义流程的生命周期界限。

| 节点名称 | 内部类型 | 功能描述 |
| :--- | :--- | :--- |
| **开始** | `start` | 流程唯一起点，负责表单初始化与变量注入。 |
| **结束** | `end` | 流程终点，执行状态结算与历史审计生成。 |

---

## 🚀 编排逻辑

1. **类型映射**：前端拖拽组件的类型会严格映射到后端 `easy-workflow` 引擎的转换函数中。
2. **属性驱动**：节点的核心配置存储在 `Properties` 字段，决定了具体行为（如审批人是谁、执行哪个脚本）。
3. **连线路由**：节点间的 `Edges` 定义了流向，是流程逻辑的物理表达。

> [!IMPORTANT]
> 每一个节点的设计都与后端引擎深度绑定，确保了「所见即所得」的执行体验。
