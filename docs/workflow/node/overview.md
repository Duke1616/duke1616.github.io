# 🧩 节点总览

工单流程由一系列功能各异的节点构成。通过拖拽这些节点，您可以像搭积木一样，快速地将业务逻辑转换为自动化的工作流。

## 📍 节点分类

根据功能划分，节点主要分为以下三类：

### 1. 生命周期 (起点与终点)
定义完整流水线的边界。

| 节点名称 | 图标 | 功能描述 |
| :--- | :--- | :--- |
| **开始节点** | 🛫 | 流程的唯一入口，初始化数据“口袋”。 |
| **结束节点** | 🏁 | 流程的终点，结算工单状态并封存历史。 |

### 2. 业务执行 (干活的节点)
完成具体的审批或运维操作，决定流程执行的 **“内容”**。

| 节点名称 | 图标 | 功能描述 |
| :--- | :--- | :--- |
| **用户节点** | 👤 | **离不开人**：由专门的人来审批、抄送或录入信息。 |
| **自动化节点**| ⚙️ | **无人值守**：自动运行脚本，处理服务器重启、扩容等苦力活。 |
| **群通知节点**| 📢 | **生态互动**：把进度实时同步到飞书群，大家都能看见。 |


### 3. 智能路由 (控制方向)
决定流程执行的 **“路径”**，就像十字路口。

| 节点名称 | 图标 | 功能描述 |
| :--- | :--- | :--- |
| **网关节点** | 🔀 | **分流与汇聚**：决定是走 A 路径还是 B 路径，或者两条路一起走。 |

---

## 🚀 编排逻辑

1. **拖拽即所得**：前端画出的流程图，会精准地翻译成后端的执行逻辑。
2. **数据口袋 (Variable Pool)**：数据在节点之间通过“口袋”传递。前面的节点放进去，后面的节点就能掏出来用。
3. **连线路由**：箭头指到哪，流程就走到哪。

> [!IMPORTANT]
> 每一个节点的设计都与后端引擎深度绑定，确保了「所见即所得」的执行体验。
