# 节点总览

工单流程由一系列功能各异的节点构成。通过可视化设计器，您可以将业务逻辑编排为自动化的工作流。

## 节点分类

根据功能划分，节点主要分为以下三类：

### 1. 生命周期
定义流程的起始与终结。

| 节点名称 | 功能描述 |
| :--- | :--- |
| **开始节点** | 流程的唯一入口，负责初始化全局变量。 |
| **结束节点** | 流程的终点，结算工单状态并封存流水记录。 |

### 2. 业务执行
执行具体的审批、自动化或通知操作，决定流程执行的具体内容。

| 节点名称 | 功能描述 |
| :--- | :--- |
| **用户节点** | **人工处理**：由特定的人员进行审批、抄送或信息录入。 |
| **自动化节点**| **自动运行**：触发运维脚本，完成分布式原子任务。 |
| **群通知节点**| **消息推送**：将流程进度实时同步至协作群聊（如飞书群）。 |


### 3. 路由控制
决定流程执行的流转路径与逻辑。

| 节点名称 | 功能描述 |
| :--- | :--- |
| **网关节点** | **逻辑分流**：根据预设条件决定流程流向，支持分流、汇聚等逻辑。 |

---

## 编排逻辑

1. **可视化编排**：前端设计器的图形化表示会精确转换为后端的执行引擎逻辑。
2. **上下文变量**：数据在节点间通过全局变量池传递。上游节点写入的数据，下游节点可直接引用。
3. **连线路由**：节点间的连线定义了任务的流转顺序与条件分支逻辑。

> [!IMPORTANT]
> 每一个节点的设计都与后端引擎深度绑定，确保了「所见即所得」的执行体验。
