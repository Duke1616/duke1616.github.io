# 📢 群通知节点

群通知节点专门用于将工单的实时进度推送到 **飞书** 或 **企业微信** 群聊。它是一个“只发消息不等人”的节点。

## 🏗️ 怎么拉群？

您可以选择两种不同的群聊模式：

1. **现有群组**：将消息发到您已经在系统里注册好的群（如：“运维报警群”、“大促发布通知群”）。
2. **自动建群**：流程运行到这一步时，系统会自动创建一个新的协作群。
    - **谁进群？**：您可以按照“审批人规则”来设置哪些相关干系人（如：产品、研发、值班人员）被自动拉入群聊。

## 🖋️ 消息里发什么？

您可以自由勾选卡片中展示的内容板块：

| 勾选项 | 对应内容 |
| :--- | :--- |
| **基础数据** | 发起人在最开始填写的申请信息。 |
| **执行结果** | 之前自动化脚本运行后的关键输出（如：发布结果摘要）。 |
| **审批历史** | 前面几个环节审批人留下的意见和处理记录。 |

## 💡 常见用法

- **开工广播**：流程刚启动，就发个消息让整个部门知道“有人开始搞高风险发布了”。
- **结果同步**：数据库变更完了，自动把变更后的具体 SQL 摘要推送到审计群。
- **故障紧急协同**：检测到故障 -> 自动创建群聊并拉入当前值班人员。

---

> [!NOTE]
> **自动办结**：这个节点属于“广播型”。消息一旦发出去，它就立即收工，流程会毫不停留地冲向下一个节点。
