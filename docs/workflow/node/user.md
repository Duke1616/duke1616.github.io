# 👤 用户节点

用户节点用于实现流程中的人工处理环节，支持 **审批模式** 与 **抄送模式**。

![节点预览](/images/node/user/overview.png)

## 1. 审批模式

您可以灵活配置审批的通过逻辑：

- **普通审批**（或签）：指定的审批人员中，只要有一人通过，流程即可向下流转。
- **会签模式**（票签）：所有指定的审批人员均需通过，流程方可向下流转。

## 2. 人员分配规则

点击“添加规则”配置处理人，支持多种策略叠加：

| 规则名称 | 逻辑说明 | 适用场景 |
| :--- | :--- | :--- |
| **指定成员** | 直接指定一个或多个固定人员。 | 固定的审批责任人 |
| **直属领导** | 自动查找工单发起人的直接上级。 | 权限申请、常规审批 |
| **分管领导** | 查找组织架构中更高级别的管理者。 | 高风险变更审批 |
| **工单发起人** | 流程回流至申请人。 | 确认结果、补充信息、被驳回重填 |
| **模板字段** | 根据申请单中填写的动态字段（如“项目负责人”）找人。 | 灵活业务场景 |
| **目前值班** | 自动匹配当前排班表中的在岗人员。 | 紧急故障处理、机房值守 |
| **指定部门** | 发送到指定部门的全员。 | 跨部门协作、行政知会 |
| **绑定团队** | 根据资产归属或业务分组自动找人。 | 服务器、数据库变更审批 |

## 3. 字段权限与数据合并

在用户节点，您可以控制处理人对表单字段的可编辑性。

<img src="/images/node/user/form.png" style="width: 100%; border-radius: 8px; margin: 16px 0;" alt="表单配置" />

### 字段权限：
- **必填**：处理人需录入该项内容方可提交。
- **只读**：仅作参考展示，不可修改。
- **隐藏**：对处理人不可见。

### 数据流转：后续合并
- **逻辑**：开启“合并”后，处理人在当前节点录入的数据将保存至工单数据池中。
- **场景**：例如运维人员审批时录入的“临时密码”，可供后续的“自动化节点”或“通知节点”直接引用。

## 4. 抄送模式

开启抄送功能后，节点将变为非阻塞状态：
- **异步提醒**：节点被触发时，抄送人会收到飞书消息。
- **即时流转**：流程不会停留在当前节点，消息发出后立即向下执行。

---

> [!TIP]
> **说明**：一个节点可以配置多条人员规则，系统会自动合并所有符合条件的审批人，并剔除重复记录。
