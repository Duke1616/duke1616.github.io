# ⚖️ 审批与抄送节点 (User Node)

用户节点是流程中人工介入的核心。根据配置不同，它可以作为 **审批环节** 或 **抄送环节**。

## 👥 审批模式

通过 `is_cosigned` 字段区分：
- **普通审批**：或签模式，任意一人处理即通过。
- **会签 (Cosign)**：所有指定的审批人必须全部同意。

## 🎯 多样化的人员规则 (Assignees)

系统提供了极度灵活的人员匹配策略，支持以下规则：

| 规则 (Rule) | 描述 |
| :--- | :--- |
| **指定人员 (appoint)** | 直接选择系统中的特定用户或角色。 |
| **工单发起人 (founder)** | 流程回流到发起人（如：确认执行结果）。 |
| **模板字段 (template)** | 从表单字段中动态提取（如：申请单中填写的「直属领导」ID）。 |
| **部门领导 (leaders)** | 自动查找发起人所属部门的主管。 |
| **分管领导 (main_leader)** | 查找组织架构中更高层级的负责领导。 |
| **值班人员 (on_call)** | 关联 **排班系统**，动态路由到当前正在值班的人员。 |
| **团队人员 (team)** | 路由到指定的资源维护团队（Team）。 |

## 📢 抄送模式 (IsCC)

当勾选 `is_cc` 时，该节点变为抄送节点：
- **行为**：启动时触发 `EventCarbonCopy`。
- **特性**：系统会自动通知相关人员，但流程会立即向下流转，不会等待人工点击通过。

## 📝 字段操作
- **变量篡改/追加**：审批人在处理时可以填写特定字段，这些数据会合并到工单数据中供后续使用。

---

> [!NOTE]
> 审批节点支持多条分配规则并行，系统会取所有规则产生的用户并集。
