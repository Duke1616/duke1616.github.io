# 💡 使用案例 (Best Practices)

本页介绍如何利用工单系统的节点编排功能，解决实际运维场景中的标准化与自动化问题。

## 🛡️ 场景一：MySQL 权限申请流程
**痛点**：权限申请口头化，缺乏审计，手动执行易出错。

### 流程编排：
1. **开始节点**：关联「权限申请表单」（包含：库名、表名、权限等级、到期时间）。
2. **条件节点**：判断权限等级。
   - 若为 `只读`：流向「组长审批」。
   - 若为 `读写`：流向「组长审批」->「DBA 审批」。
3. **执行节点 (Task)**：审批通过后，触发 ETask 脚本，在数据库自动创建账号并赋权。
4. **抄送节点**：将临时账号密码通过隐私消息发送给申请人。

---

## ☁️ 场景二：云服务器自动扩容
**痛点**：大促期间资源紧张，人工扩容响应慢。

### 流程编排：
1. **开始节点 (API 触发)**：由 EAlert 监控告警自动触发，传入 `instance_id`。
2. **审批节点**：运维负责人点击「确认扩容」。
3. **执行节点**：调用云厂商 API 执行实例规格升级。
4. **结束节点**：记录执行日志并归档。

---

## 🛠️ 场景三：代码上线发布
**痛点**：核心业务发布需要多方会签，且需在特定时间执行。

### 流程编排：
1. **开始节点**：填写发布版本号、上线说明。
2. **审批节点 (会签)**：研发组长 + 测试组长 **全部同意** 方可继续。
3. **执行节点 (定时)**：在预定的发布窗口自动触发 Jenkins Pipeline。
4. **条件节点**：校验执行结果。
   - 成功：抄送全员通知群。
   - 失败：自动触发「回滚节点」。

---

> [!TIP]
